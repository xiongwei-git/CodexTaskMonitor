# AgentTaskMonitor collaboration guide

## Start here

Before product, research, design, or implementation work:

1. Read `PROJECT_MEMORY.md`.
2. Read `docs/product-discovery.md`.
3. Inspect only the decision or research files relevant to the current question.

## Working rules

- Treat this as an ongoing product-incubation project.
- Keep confirmed facts, user statements, assumptions, and recommendations clearly separated.
- Do not turn name-based guesses into product requirements.
- Prefer one focused discovery question at a time while the concept is still forming.
- Record durable product decisions in `PROJECT_MEMORY.md` and, when substantial, in `decisions/`.
- Put source-backed competitor or market findings in `research/`.
- Do not begin implementation until the problem, first user, MVP hypothesis, and acceptance signal are explicit.

## Project state

- `PROJECT_MEMORY.md` holds current project state and handoff context.
- `README.md` is the human-facing project entry.
- `docs/product-discovery.md` is the living product-discovery canvas.
