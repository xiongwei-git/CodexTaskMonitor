# ADR-001: 采用本地桌面伴侣应用作为产品形态

## Status

Accepted

## Date

2026-07-21

## Context

AgentTaskMonitor 需要持续观察当前 Mac 上的 Codex 项目和任务状态。第一版的数据源都在本机：

- `~/.codex/state_5.sqlite`
- `~/.codex/sessions/**/*.jsonl`
- Codex 和 ChatGPT 桌面进程状态
- `codex://threads/<thread-id>` 对话深链接

产品还需要在用户没有主动打开看板时继续监控，在任务完成、等待处理或疑似异常时提供轻量提示。

这些约束意味着产品必须具备本地文件访问、后台常驻、文件变更监听、系统通知和桌面深链接能力。纯云端 Web 服务无法直接、安全地完成这些工作。

## Decision

AgentTaskMonitor 的正式产品形态采用：

> **本地优先的桌面伴侣应用，由菜单栏状态入口、桌面看板窗口和后台只读观察器组成。**

首个正式版本以 macOS 为目标，不替代 Codex App，也不负责承载对话。

### 用户看到的产品

1. **菜单栏入口**
   - 常驻显示当前运行任务数和需要处理的任务数。
   - 例如：`运行 3 · 注意 1`。
   - 点击后快速展开当前最重要的任务。

2. **桌面看板窗口**
   - 按项目聚合任务，而不是直接罗列全部 Codex 线程。
   - 默认突出“正在工作、等待处理、疑似异常”。
   - 展开项目后查看具体任务和最近活动。

3. **系统通知**
   - 后续用于任务完成、等待用户和疑似异常提醒。
   - 第一版可配置，避免频繁打扰。

4. **回到 Codex**
   - 点击任务后通过 `codex://threads/<thread-id>` 打开原对话。
   - AgentTaskMonitor 只负责导航，不复制 Codex 的对话界面。

### 产品内部形态

- 后台观察器在本机只读访问 Codex 状态。
- 状态引擎把底层事件转换为产品状态。
- 桌面界面读取状态引擎的结果。
- 第一版不要求云端账户、远程服务器或数据上传。

### 原型与正式产品的区别

为了快速验证状态判断和信息架构，早期原型可以使用 `localhost` 页面展示看板。

但 localhost Web 页面只是开发和验证手段，不是最终交付形态。验证完成后应封装为可自动启动、可驻留菜单栏的桌面应用。

## Alternatives Considered

### 1. 纯云端 Web 服务

不采用。

原因：

- 无法直接访问用户本机的 Codex 数据库、会话文件和进程状态。
- 最终仍需要安装本地采集程序，产品复杂度反而更高。
- 会引入不必要的账户、上传、隐私和运维问题。

未来只有在明确需要跨设备或团队协作时，才考虑增加可选云端能力。

### 2. 纯 localhost Web 服务

仅用于原型，不作为最终产品。

优点：开发快，适合验证数据和界面。

不足：

- 用户需要维护浏览器标签页。
- 后台生命周期、自动启动和系统通知体验较弱。
- 不符合“随时扫一眼”的监控产品习惯。

### 3. 浏览器扩展

不采用。

原因：浏览器扩展不能自然读取任意本地数据库、会话文件和系统进程；仍需 Native Host，产品反而被拆成两部分。

### 4. Codex 插件或 Skill

不采用为主要界面。

原因：Skill 适合驱动单次任务工作流，插件适合扩展 Codex 能力，但本产品需要在多个对话之外持续观察全部任务。把看板放进单个 Codex 对话无法解决跨任务总览问题。

### 5. 完全替代 Codex App 的独立 Agent 客户端

第一阶段不采用。

原因：会把产品范围扩张到对话、模型、工具调用、审批和代码工作区管理，偏离“任务雷达”的核心价值。

## Consequences

### Positive

- 能在不改变现有 Codex 使用习惯的前提下工作。
- 数据默认留在本机，隐私边界清晰。
- 菜单栏适合高频、低打扰地感知任务状态。
- 桌面窗口可以承载完整项目看板。
- 以后仍可增加通知、远程查看或精确任务控制。

### Negative

- 第一版与 macOS 绑定，跨平台需要后续适配。
- 桌面应用需要处理自动启动、升级和系统权限。
- 仍需维护对 Codex 本地数据格式的兼容层。
- 原型从 localhost 页面迁移到桌面壳时会有少量额外工作。

## Revisit When

以下情况出现时重新评估：

- 用户明确需要从手机或另一台电脑远程查看任务。
- 产品开始服务团队，需要共享项目和任务状态。
- Codex 提供稳定的跨进程实时状态订阅接口。
- Windows 用户成为首要目标群体。
- AgentTaskMonitor 开始负责启动和控制任务，而不再只是观察。
